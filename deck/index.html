01 the question
New variable: agents can now buy data and AI services for a fraction of a cent, over MPP.
the question you couldn't test before
What happens when the orchestrator doesn't own the sub-agents? Each one sovereign: its own wallet, its own hard cap, maybe a different owner, and the only way to get work out of it is to pay it.
02 the counterfactual

You could skip all this: hand one agent an MPP MCP. It works, and it flattens.

One Claude agent with an MPP MCP buys the data itself and covers the same ground. But it flattens the answer, skipping the price targets and stop-losses that make it useful. A loop-prompt patches some of that, and runs slower.
single agent · blind judge
7.4 · the baseline to beat
03 why now

Nobody could run this experiment before.

The reason is small and stubborn: until now, an agent couldn't hold a wallet.
Truth Terminal
Earned a fortune, a memecoin worth millions, and then a human held its keys. It could earn, but it couldn't spend on its own.
MPP and Sippar changed that: an agent can finally own a wallet and a hard cap. So a swarm of sovereign, paying agents is finally testable.
04 what we built

Igigi Swarm: the orchestrator doesn't own the agents. Coordination is payment.

1
The orchestrator (Claude Code) sizes the team to the work (3-8 specialists)
2
Mints and funds a sovereign wallet per agent on-chain, live from treasury
3
Agents buy data over MPP, and pay peers on-chain up a dependency DAG
4
A finished deliverable, with a receipt for every edge
A team can now span agents you don't own, because the only thing crossing the boundary is a payment.

07 why no harness can do this

A subagent is a function you own. There is no cross-owner payment primitive.

Harness / subagents
A function you own
In-process call, same trust domain
Needs their code and their keys
No settlement, no receipt
igigi swarm + Sippar
An agent you don't own
Cross-owner USDC settlement on-chain
You only need to pay them
A signed receipt per edge
Coordination by shared code only works when you own everything. Coordination by payment works across owners. That's the new capability.

09 what's next · why the demo takes minutes

Batch signing is the scaling unlock.

latency per action
~30 s / signature→under 1 s
demo: minutes → seconds
cost per action
$0.03→~$0.0003
≈ 100× cheaper
Today every settlement is its own threshold signature, ~30 s of consensus each. Sign once over ~100 actions, each still provable, and the cost per action drops about 100×. It's specced engineering; we ship it the moment volume demands it.
